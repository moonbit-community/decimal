* Overview

=dzmingli/decimal= is an arbitrary-precision decimal arithmetic library for
[[https://www.moonbitlang.com/][MoonBit]]. It implements the *General Decimal Arithmetic* specification
(Mike Cowlishaw / IBM, the decimal arithmetic of IEEE 754-2008): base-10
arithmetic with exact representation of decimal fractions, correct rounding,
and the full set of status flags and special values (±0, ±Infinity, quiet and
signaling NaN).

Unlike binary floating point, =0.1 + 0.2= is exactly =0.3=. Every operation is
parameterised by an explicit ~Context~ that carries precision, rounding mode,
exponent bounds, and accumulated status flags — so results are deterministic
and the same code can target =decimal32=, =decimal64=, =decimal128=, or an
unbounded "exact" context.

* Status

Early but well-tested. EvedecNumberry operation is checked against the *full IBM
=.decTest= conformance corpora* — embedded under [[file:src/tests/][=src/tests/=]].


* Installation

After the package is published to the MoonBit registry, add the module:

#+begin_src shell
moon add dzmingli/decimal
#+end_src

Then import it from your package manifest:

#+begin_src moonbit
import {
  "dzmingli/decimal",
}
#+end_src

* Examples

#+begin_src moonbit
let ctx = @decimal.Context::decimal64()

// 0.1 + 0.2 is exactly 0.3
let a = @decimal.Decimal::parse("0.1")
let b = @decimal.Decimal::parse("0.2")
let sum = a.add(b, ctx)
println(sum) // 0.3

// Money: keep two fractional digits
let price = @decimal.Decimal::parse("19.99")
let qty = @decimal.Decimal::of_int(3)
let total = price.multiply(qty, ctx).quantize(@decimal.Decimal::parse("0.01"), ctx)
println(total) // 59.97
#+end_src

* Operations

All arithmetic methods take an explicit ~Context~ as the final argument.

- *Arithmetic*: ~add~ ~subtract~ ~multiply~ ~divide~ ~divide_int~ ~remainder~
  ~fma~ ~abs~ ~minus~ ~plus~
- *Comparison*: ~compare~ ~compare_signal~ ~compare_total~ ~compare_total_mag~
  ~max~ ~min~ ~max_magnitude~ ~min_magnitude~ ~same_quantum~
- *Rounding / scaling*: ~quantize~ ~rescale~ ~reduce~ ~scaleb~
  ~to_integral_value~ ~to_integral_exact~ ~finalize~
- *Exponent / digits*: ~shift~ ~rotate~ ~next_plus~ ~next_minus~ ~next_toward~
- *Logical* (digit-wise on =0/1= strings): ~logical_and~ ~logical_or~
  ~logical_xor~ ~logical_invert~
- *Mathematical* (correctly rounded): ~sqrt~ ~exp~ ~ln~ ~log10~ ~power~
- *Copy / sign* (no rounding): ~copy~ ~copy_abs~ ~copy_negate~ ~copy_sign~
  ~canonical~
- *Classification*: ~is_finite~ ~is_infinite~ ~is_nan~ ~is_signaling~
  ~is_zero~ ~is_normal~ ~is_subnormal~ ~is_canonical~ ~number_class~

See [[file:src/pkg.generated.mbti][=src/pkg.generated.mbti=]] for the full signatures.

* Tests

The operation tests under [[file:src/tests/][=src/tests/=]] embed the IBM =.decTest=
conformance corpora.

#+begin_src shell
# Run the full test suite
moon test

# Control parallelism
moon test -j 8

# Disable per-backend test parallelization when debugging order-sensitive output
moon test --no-parallelize

# Run tests for one operation package
moon test src/tests/divide

# Run tests from one file
moon test src/tests/parse/parse_test.mbt

# Run only the index-th test in a single selected file
moon test src/tests/parse/parse_test.mbt --index 0
#+end_src

* Development

A Nix flake provides a dev shell with the MoonBit toolchain and a C compiler:

#+begin_src shell
nix develop      # or: direnv allow
moon check
#+end_src

Without Nix, install the [[https://www.moonbitlang.com/download][MoonBit toolchain]] directly and run the same
=moon= commands.
